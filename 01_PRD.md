1. EXECUTIVE SUMMARY
The objective of this project is to design, develop, deploy, and hand over a professional institutional website for the NIE Technology Business Incubator (NIETBI).
The platform will serve as the official digital presence of NIETBI and act as a centralized portal for startup incubation activities, program information, startup showcases, events, institutional announcements, downloads, donations, and stakeholder engagement.
The website shall reflect the academic credibility and institutional identity of The National Institute of Engineering while promoting entrepreneurship, innovation, research commercialization, and startup ecosystem development.
The platform shall be mobile-first, secure, scalable, easy to maintain, and designed for long-term institutional use.
________________________________________
2. PROJECT OBJECTIVES
The website shall:
•	Establish NIETBI's official online presence.
•	Showcase incubated and graduated startups.
•	Present incubation programs and opportunities.
•	Publish news, events, achievements, and announcements.
•	Provide access to institutional policies and downloadable documents.
•	Accept startup enquiries and incubation applications.
•	Support donations and social impact initiatives.
•	Enable NIETBI administrators to update content independently.
•	Ensure a consistent visual identity aligned with NIE branding.

3. DESIGN GOVERNANCE FRAMEWORK
3.1 Design Philosophy
The website shall follow the institutional design language of The National Institute of Engineering (NIE).
The overall visual identity shall communicate:
•	Academic Excellence
•	Innovation & Entrepreneurship
•	Institutional Trust
•	Government-Backed Credibility
•	Professionalism
•	Long-Term Sustainability
The platform shall not resemble:
•	Startup SaaS dashboards
•	Corporate marketing websites
•	Agency portfolio websites
•	Highly animated landing pages
The design shall position NIETBI as:
"The Innovation and Entrepreneurship Arm of NIE."
________________________________________
3.2 Visual Standards
Layout
•	Clean and structured layouts
•	Content-first approach
•	Consistent visual hierarchy
•	Professional spacing system
•	Card-based information architecture
•	Minimal and purposeful animations
Color Strategy
Primary Theme:
•	Institutional Navy Blue
Secondary Theme:
•	NIE Orange
Supporting Colors:
•	White
•	Light Gray
•	Neutral Text Colors
Orange shall primarily be used for:
•	Calls-to-action
•	Buttons
•	Statistics
•	Highlights
•	Links
________________________________________
3.3 Responsive Design
The website shall be designed using a mobile-first methodology.
Supported Devices:
•	Mobile Phones
•	Tablets
•	Laptops
•	Desktop Systems
The website shall function consistently across modern browsers.

5. CONTENT MANAGEMENT SYSTEM (CMS)
5.1 Administrative Access
A secure administrative portal shall be provided.
Access shall be restricted to authorized NIETBI personnel through secure authentication.
Administrative Route:
/admin
________________________________________
5.2 CMS Philosophy
The CMS shall allow content updates while preserving website design integrity.
Administrators shall manage content.
Administrators shall not modify the website design structure.
________________________________________
5.3 Startup Management Module
Administrators can:
•	Add Startup
•	Edit Startup
•	Delete Startup
Fields:
•	Startup Name
•	Logo
•	Sector
•	Description
•	Founder Name
•	Website Link
•	Graduated Status
________________________________________
5.4 Event Management Module
Administrators can:
•	Add Event
•	Edit Event
•	Delete Event
Fields:
•	Event Title
•	Event Date
•	Description
•	Banner Image
•	Registration Link
•	Event Status
Statuses:
•	Upcoming
•	Completed
________________________________________
5.5 Blog Management Module
Administrators can:
•	Add Blog
•	Edit Blog
•	Delete Blog
Fields:
•	Title
•	Author
•	Publish Date
•	Featured Image
•	Content
Content shall be managed through a controlled Markdown editor.
The editor explicitly restricts:
•	Custom HTML
•	Inline CSS
•	Font Changes
•	Custom Color Styling
•	Embedded Scripts
These restrictions ensure adherence to the approved NIETBI/NIE brand guidelines and maintain visual consistency throughout the platform.
________________________________________
5.6 Downloads Management Module
Administrators can:
•	Upload Documents
•	Update Documents
•	Remove Documents
Fields:
•	Title
•	Description
•	Category
•	PDF File
Document Categories:
•	Policies
•	Reports
•	Templates
•	Forms
•	Guidelines
Maximum File Size:
10 MB per document
Supported Format:
PDF only
________________________________________
5.7 Contact Settings Module
Administrators can update:
•	Contact Numbers
•	Email Addresses
•	Office Hours
•	Social Media Links
•	Application Status
No code deployment shall be required for these updates.
________________________________________
5.8 CMS Governance
The CMS enables content management only.
Editable:
•	Startups
•	Events
•	Blogs
•	Downloads
•	Contact Information
•	Social Media Links
•	Application Status
Non-Editable:
•	Website Layout
•	Navigation Structure
•	Page Hierarchy
•	Color Palette
•	Typography
•	Design System
•	UI Components
•	Animations
Changes to these items shall be treated as separate design or development requests.

6. APPLICATION MANAGEMENT
Startup Application Form
Applicants shall submit:
•	Startup Information
•	Founder Details
•	Contact Information
•	Startup Stage
•	Sector
•	Pitch Deck Link
Direct file uploads shall not be accepted.
Applicants shall provide publicly accessible cloud-storage links.
A mandatory confirmation checkbox shall require applicants to verify link accessibility.
________________________________________
Contact Form
General enquiries submitted through the website shall be routed to the configured institutional email through a secure serverless transactional email service.
Supported providers may include:
•	Resend
•	SendGrid
•	Equivalent transactional email providers
This project includes form routing only.
The project does not include:
•	Email Hosting
•	Google Workspace Setup
•	Microsoft 365 Setup
•	Institutional Mail Server Provisioning
